 - Будучи успешно инициализированным, сервер готов принимать соединения от клиентов на порт обратной связи.
 - После принятия сервером соединения от клиента между ними начинается взаимодействие, состоящее из передачи сообщений.

 - Сообщение - последовательность частей сообщения, завершаемая символом U+001E.
 - Части сообщения внутри сообщения разделены между собой одинарными символами U+001F.
 - Часть сообщения - последовательность символов, не содержащая символов U+001E и U+001F.

 - Сообщения от клиента к серверу выполняют роль команд, на которые сервер даёт ответы.
 - Первая часть сообщения в команде определяет её тип, последующие являются её аргументами.
 - Если сервер получает команду, не известную ему, то он отправляет клиенту ответ "unknown_command".
 - Если сервер получает команду с неправильным количеством аргументов, то он отправляет клиенту ответ "invalid_args_count".
 - Тайм-аут неактивности клиента (отсутствия посылаемых им команд) - 30 мин.

 - Определены следующие команды:
 1) Логин: login <name> <password>
 - Получив имя <name> и пароль <password> пользователя, сервер проверяет их корректность и аутентичность.
 - Ответ: login_invalid - в случае неуспеха, login_valid - в случае успеха.
 - Эффект: в случае успеха - текущее соединение закрепляется за переданным именем пользователя.
 2) Логаут: logout
 - Ответ: logout_ack
 - Эффект: Текущее соединение не закреплено ни за каким именем.
 3) Отключение: disconnect
 - Ответ: disconnect_ack
 - Эффект: Соединение закрыто сервером.
 4) Продление тайм-аута ожидания команд: prolongate
 - Ответ: prolongate_ack
 - Эффект: Тайм-аут неактивности клиента сброшен.
 5) Инвалидация других соединений: invalidate
 - Ответ: invalidate_ack
 - Эффект: по всем соединениям, закреплённым за тем же именем, что и текущее соединение, кроме него самого послано оповещение инвалидации.

 - Сервер может отправлять клиенту сообщение по собственной инициативе.
 - Такие сообщение можно разделить на две группы: оповещения и сообщения о событии.

 - Определены следующие оповещения:
 1) Тайм-аут - "timeout". Отправляется в случае тайм-аута неактивности клиента.
 - После этого оповещение сервер закрывает соединение.
 2) Инвалидация - "invalidated". Отправляется в любом из двух случаев:
	* Получение по какому-либо другому соединению, закреплённому за тем же именем, что и текущее, команды "invalidate".
	* Изменение внешним образом пароля пользователя, за чьим именем закреплено текущее соединение.
 - После этого оповещение текущее соединение считается не закреплённым ни за каким именем.
 3) Ошибка ввода/вывода - "io_error". Отправляется в случае ошибки при использовании сервером текущего соединения.
 - После этого оповещение сервер закрывает соединение.
 4) Остановка сервера - "shutdown". Отправляется в случае завершения сервером своей работы.
 - После этого оповещение сервер закрывает соединение.

 - Сообщение о событии отправляется при наступлении какого-либо события для пользователя, за именем которого закреплено текущее соединение:
 - Сообщение о событии состоит из следующий частей сообщения:
	* "event"
	* Локальное время, на которое было зарегистрировано данное событие.
	- Формат представления - количество миллисекунд, прошедших с начала эпохи до данного времени как локального в зоне GMT.
	* Текст события.
