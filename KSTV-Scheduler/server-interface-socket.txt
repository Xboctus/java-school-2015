 - Сервер готов принимать соединения от клиентов на определённый порт. (Этот порт может быть получен с помощью действия get_port.)
 - Получив соединение, сервер ждёт по нему от клиента логина - три строки:
	* "login"
	* Имя пользователя
	* Пароль пользователя
 - Таймаут ожидания логина - 10 сек.
 - При истечении таймаута ожидания логина сервер отправляет клиенту строку "login_timeout" и закрывает соединение.
 - Если первая полученная от клиента строка - не "login", сервер отправляет клиенту строку "login_expected" и закрывает соединение.
 - Получив имя и пароль пользователя, сервер проверяет их валидность. В случае невалидности, сервер отправляет клиенту строку "login_invalid" и закрывает соединение.
 - Если пара имя-пароль валидна, то сервер отправляет клиенту строку "login_valid" и оставляет соединение открытым. С этого момента оно считается закреплённым за полученным логином.
 - При наступлении какого-либо события для какого-либо пользователя, сервер посылает по всем соединениям, закреплённым за логином этого пользователя, три строки:
	* "event"
	* Локальное время, на которое было зарегистрировано данное событие. Формат представления - количество миллисекунд, прошедших с начала эпохи до данного времени как локального в зоне GMT.
	* Текст события.
 - Начальный таймаут активности клиента (с момента отправки "login_valid") - 30 мин.
 - Получив от клиента строку "prolongate", сервер отправляет ему строку "prolongate_ack" и сбрасывает таймаут его активности. (Т.е. у клиента есть ещё 30 мин. для проявления активности.)
 - При истечении таймаута активности клиента сервер отправляет ему строку "timeout" и закрывает соединение.
 - Получив от клиента строку "logout", сервер отправляет ему строку "logout_ack" и закрывает соединение.
 - При любой ошибке, связанной с использованием соединения, сервер пытается отправить по нему строку "error" и закрывает его.
 - Любая передаваемая строка всегда считается оканчивающейся первым же от её начала символом перевода строки. (Это символ не является её частью.)
