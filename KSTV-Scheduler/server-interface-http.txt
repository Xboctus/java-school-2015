URL'ы:
	/event_port - порт обратной связи
	/users - пользователи
	/users/<name> - пользователь с именем <name>
	/users/<id> - пользователь с идентификатором <id>
	/users/:me - текущий пользователь
	/users/<ref>/events - события пользователя <ref> (<ref> - <name>, <id> или :me)
	/users/<ref>/events/<id> - событие пользователя <ref> с идентификатором <id>
	/events - события
	/events/<id> - событие с идентификатором <id>

- Формат принимаемых и отправляемых сервером сообщений по HTTP такой же, как по сокету, с тем исключением, что ограничитель сообщения необязателен.

= (Описание действия) =
Доступ - HTTP-метод и путь в URL (отн. /<App>/<Servlet>), используемые клиентом для совершения действия
Клиент - параметры, передаваемые клиентом серверу
Код состояния - код состояния HTTP, возвращаемый сервером клиенту
Сервер - параметры, возвращаемые сервером клиенту
Состояние сессии - состояние сессии с клиентом после выполнения запроса

= (Для всех действий) =
Коды состояния:
	404 Not found - путь имеет неправильный формат
	400 Bad Request - лексическая ошибка (лексически некорректное имя пользователя или пароль, ошибка в формате параметров и т.п.)

= Анонимные действия =
(Активная сессия не требуется.)
Состояние сессии: не изменяется

== Получение порта сервера для обратной связи ==
Доступ: GET /event_port
Код состояния:
	200 OK - запрос выполнен успешно
Сервер:
	<port> - прослушиваемый сервером порт (0, если порт недоступен)

== Добавление пользователя ==
Доступ: POST /users
Клиент:
	<name> - имя пользователя
	<password> - пароль пользователя
	<timezone> - временная зона пользователя
	<active> - статус пользователя
Код состояния:
	201 Created - пользователь создан
	409 Conflict - пользователь с таким именем уже есть

= Действия изменения состояния сессии =

== Логин ==
Доступ: PUT /login
Клиент:
	<name> - имя пользователя
	<password> - пароль пользователя
Код состояния:
	200 OK - аутентификация пройдена
	401 Unauthorized - иначе
Состояние сессии:
	если аутентификация пройдена - устанавлена с привязкой на имя <name>
	иначе - не изменяется

== Логаут ==
Доступ: PUT /logout
Код состояния:
	200 OK - ошибок нет
Состояние сессии: не устанавлена

= Неанонимные действия =
(Требуется активная сессия.)
Код состояния:
	401 Unauthorized - у клиента нет активной сессии
Состояние сессии: не изменяется

== Запрос информации о себе ==
Доступ: GET /users/:me
Код состояния:
	200 OK - запрос выполнен успешно
Сервер:
(параметры передаются только если код состояния 200 OK)
	<timezone> - временная зона пользователя
	<active> - статус пользователя
	<event_count> - количество событий у пользователя

== Изменение информации о себе ==
Доступ: PUT /users/:me
Клиент:
	<old_password>
	<new_password>
	<new_timezone>
	<new_active>
Код состояния:
	200 OK - ошибок нет

== Запрос списка своих событий ==
Доступ: GET /users/:me/events
Код состояния:
	200 OK - запрос выполнен успешно
Сервер:
(параметры передаются только если код состояния 200 OK)
	<event_count> - количество событий у пользователя
	<datetime_1> - дата и время 1-го события
	<text_1> - текст 1-го события
	<datetime_2> - дата и время 2-го события
	<text_2> - текст 2-го события
	...

== Добавление события ==
Доступ: POST /users/:me/events/
Клиент:
	<text>
	<datetime_from>
	<datetime_to>
Код состояния:
	201 Created - событие добавлено
	409 Conflict - такое событие у пользователя уже есть
(Заголовок Location содержит идентификатором добавленного события.)

== Удаление события ==
Доступ: DELETE /users/:me/events/<id>
Коды состояния:
	204 No Content - событие удалено
	404 Not found - такого события у пользователя нет

Форматы параметров:
	<name> - имя-идентификатор, непустое, первый символ - буква [a-zA-Z_], последующие - буква или цифра [a-zA-Z_0-9]
	<password> - произвольная последовательность символов
	<text> - произвольная последовательность символов
	<datetime> - дата-время в формате кол-ва миллисекунд прошедших с начала эпохи до этого времени как локального в зоне GMT
	<event_count> - целое неотрицательное число
	<port> - целое число в диапазоне от 0 до 65535 вкл.
	<timezone> - сдвиг в миллисекундах отн. GMT
	<active> - true или false
	<id> - целое неотрицательное число
